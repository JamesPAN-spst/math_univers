本文件记录 Mathematica 数学教育网站从规划到交付的完整工作流。


████████████████████████████████████████████████████████████████████████
  阶段一 · 任务分解与架构设计
████████████████████████████████████████████████████████████████████████

  1) 根据数学学科体系将知识领域划分为 7 个板块（代数、分析核心、
     微分方程&数值、几何&拓扑、概率&随机、应用数学、基础&离散），
     共 28 个模块、174 个章节。

  2) 在 index.html 中以 5 阶段路线图（Phase 0-4）呈现所有模块的
     依赖关系和学习路径；模块按 domain 着色，提供悬浮章节大纲。

  3) 将总工程拆分为独立的子任务，使每个 Agent 可并行工作、互不干扰。


████████████████████████████████████████████████████████████████████████
  阶段二 · 样板与规则建立
████████████████████████████████████████████████████████████████████████

  1) 构建共享基础设施:
     - shared/theme.css  — 全局主题变量（亮/暗模式）
     - shared/layout.css — 页面布局（topbar + toc-sidebar + main-content）
     - shared/components.css — 组件样式（step-card、inline-quiz 等）
     - shared/components.js — 交互逻辑（quiz 渲染、答案检查、滚动监听）
     - shared/storage.js  — localStorage 主题和进度存储

  2) 制定严格的页面模块化规则:
     - page_modularization_rules.txt: 定义骨架结构、CSS class 白名单、
       内容质量标准、ID 命名规范、违规检查清单。
     - 每章 4 KP × 5 教学步骤 + inline-quiz 的固定格式。

  3) 制作示范模板:
     - sample_chapter.html (groups-ch3 完整示范)
     - sample_menu.html (groups 概览页示范)

  4) 对模板进行多轮检测与重构，确保与 index.html 路由和 rules 完全一致。


████████████████████████████████████████████████████████████████████████
  阶段三 · 知识点大纲内容生成
████████████████████████████████████████████████████████████████████████

  1) 创建 knowledge_outline.txt（~33000 行）作为所有知识点的统一存储，
     按 7 Agent 区域分块，每个 KP 含 6 项子结构:
     ① MOTIVATION  ② ABSTRACTION  ③ DERIVATION
     ④ APPLICATION ⑤ EXAMPLE      ⑥ QUIZ

  2) 为 7 个 Agent 编写 .agent.md 指令文件，明确:
     - 负责模块范围、章节数、KP 总数
     - 工作流程（读参考 → 填写 → 标记 DONE → 写报告）
     - 内容质量要求（字数下限、公式数量、完整性标准）

  3) 按顺序启动 7 个 Agent，全部完成后进行数据验证:
     - 674 KP 全部填写、0 空白
     - 6 项子结构各出现 674 次
     - 168 章节全标记 DONE

  4) 对每个 Agent 的工作选取一个中间章节进行内容抽查，
     验证数学准确性、结构完整性、教学质量。


████████████████████████████████████████████████████████████████████████
  阶段四 · Agent 自审报告与审核
████████████████████████████████████████████████████████████████████████

  1) 每个 Agent 完成后自动生成 reports/agent-N-*-report.md，包含:
     - 完成概况（模块数、章节数、KP 数）
     - 各模块编排思路摘要
     - 自评（评分、最佳/待改进章节、困难记录）
     - 交叉引用建议

  2) 审核流程:
     - 阅读全部 7 份报告，提取自评中的"需改进"项
     - 验证自评中指出的具体问题（如 quiz 答案错误）
     - 对确认的错误立即修正
     - 对深度/覆盖权衡类问题，输出解决方案文档


████████████████████████████████████████████████████████████████████████
  阶段五 · 题库构建
████████████████████████████████████████████████████████████████████████

  1) 制定 quiz_construction_rules.txt，定义:
     - 非 M2 模块: 每章 25+3 题（7基础 + 10中等 + 8难 + 3应用）
     - M2+ 模块: 每章 10 道应用型选择题
     - 题目类型、难度分布、质量标准、自审报告模板

  2) 为 7 个板块分别创建题目生成 Agent:
     - 读取 knowledge_outline.txt 获取知识点内容
     - 按 rules 生成题目，写入 question_bank.txt 对应区域
     - 完成后提交 reports/quiz-agent-N-*-report.md

  3) 对每个单元抽查 ≥10 道题目，检验:
     - 计算/算法题比例 ≥ 20/25
     - 难度分布合理性
     - 数学正确性
     - 应用题贴近真实场景


████████████████████████████████████████████████████████████████████████
  阶段六 · HTML 页面批量生成
████████████████████████████████████████████████████████████████████████

  1) 编写 _build_pages.js (32KB) 批量生成引擎:
     - 读取 knowledge_outline.txt 提取各章 4 KP 的 6 项子结构
     - 读取 question_bank.txt 匹配章节题目，注入 QUESTIONS 数组
     - 按 page_modularization_rules.txt 骨架生成 168 章节 HTML
     - 生成 28 个模块概览页（三列卡片布局 + 进度条）
     - 每页包含 topbar、toc-sidebar、step-card、inline-quiz、quick-check

  2) 页面质量修复脚本链:
     - _fix_formatting.js → 修复 step-card 结构、section 嵌套
     - _fix_formatting2.js → 修复残余格式异常
     - _fix_stepcards.js → 补全缺失 step-card 容器
     - _fix_truncated.js / _fix_truncated2.js → 修复内容截断
     - _fix_remaining.js → 修复剩余边缘 case
     - _fix_v3.js → 第三轮整体修复

  3) 最终产出: 168 章节页 + 28 概览页 + 1 首页 = 197 HTML 页面

  阶段七 · 视觉设计与交互增强
████████████████████████████████████████████████████████████████████████

  1) 共享 CSS/JS 基础设施:
     - theme.css (3KB): CSS 变量定义，三套主题（亮/暗/霓虹），
       字体加载（Cormorant Garamond / Raleway / Fira Code / Noto Sans SC）
     - layout.css (10KB): 页面布局（topbar + toc-sidebar + main），
       CSS Grid 自适应，移动端响应式
     - components.css (16KB): 组件样式（step-card 渐变边框、quiz 卡片、
       wrong-book 弹窗、搜索胶囊、霓虹主题 glassmorphism 覆写）
     - components.js (14KB): quiz 渲染/答案检查、scroll-spy 目录高亮、
       wrong-book UI、浮动胶囊全文搜索、页面过渡动画
     - storage.js (9KB): localStorage 主题持久化、进度追踪、
       WrongBook 错题本、WebGL 霓虹画布

  2) 视觉特色:
     - index.html (35KB): 5 阶段路线图、domain 着色模块卡片、

     - 霓虹模式: WebGL domain-warping fBm 着色器，
       7 套余弦调色板（默认虹彩 + 6 个学科色），每板约 28s 过渡，
       全周期约 3 分钟，smoothstep 平滑切换

  3) 全文搜索系统:
     - _gen_search_index.js: 从 197 页提取正文、去 TeX 标记、
       每页上限 2000 字符，生成 search-data.js (325KB)
     - 浮动胶囊搜索框: 固定左侧，侧边栏感知，实时匹配 body 文本

════════════════════════════════════════════════════════════════════════
  核心原则
════════════════════════════════════════════════════════════════════════

  • 先规则后执行 — 所有内容生成前必须有明确的 rules 文档
  • Agent 并行化 — 不同领域 Agent 互不干扰，可并行工作
  • 双重质量保证 — Agent 自审 + 人工抽查/报告审核
  • 增量可追溯 — 每步产出有文件记录（outline、reports、method）
  • 统一格式 — 所有输出严格遵循预定义的结构和格式模板
