  阶段一 · 任务分解与架构设计
████████████████████████████████████████████████████████████████████████

  1) 根据数学学科体系将知识领域划分为 7 个板块（代数、分析核心、
     微分方程&数值、几何&拓扑、概率&随机、应用数学、基础&离散），
     共 28 个模块、174 个章节。

  2) 在 index.html 中以 5 阶段路线图（Phase 0-4）呈现所有模块的
     依赖关系和学习路径；模块按 domain 着色，提供悬浮章节大纲。

  3) 将总工程拆分为独立的子任务，使每个 Agent 可并行工作、互不干扰。

  阶段十三 · 学习反馈驱动的内容精细化 (Iterative Refinement)
████████████████████████████████████████████████████████████████████████

  方法论:
    在实际学习过程中，用户阅读页面后可能发现内容虽然正确但过于
    精炼——推导步骤不够详细、结论缺少"为什么"的解释、公式之间
    缺少过渡性文字。这类问题在批量生成阶段难以避免，因为内容生成
    Agent 缺少真实学习者的视角。

  工作流:
    1) 学习者阅读某章节，标记「读不懂 / 跳跃太大 / 结论突兀」的位置
    2) 针对标记位置，逐个知识点进行内容精细化:
       - 🎯 Motivation: 从具体例子出发，而非直接给出抽象定义
       - 🔍 Abstraction: 补充「从一元到多元如何推广」的对比说明
       - 📐 Derivation: 拆分为编号的子步骤，每步给出推导过程和直觉解释；
         添加 h4 子标题划分逻辑块；解释「为什么这个条件是必要的」
       - 🌐 Application: 从一句话扩展为具体场景描述，连接到后续课程
       - ✏️ Example: 分步解题（第 1 步、第 2 步…），列出常见错误及成因
    3) 保持数学准确性——精细化只增加解释，不改变结论
    4) 使用 HTML 结构化元素（h4、ul/li、strong/em）改善可读性

  首次应用:
    - 章节: modules/foundation/mc2/ch07.html (多元微积分初步)
    - 4 个 KP 全部精细化: 动机、抽象、推导、应用、例题各环节均扩展
    - 内容量: 从约 ~600 字 / KP 扩展至 ~1500 字 / KP
    - 重点补充: ε-δ 定义的几何解读、偏导≠连续的反例链、
      梯度垂直等值线的证明、Hessian 判别式 AC-B² 的代数由来、
      隐函数定理的圆的直觉例子、两个定理的互推关系

  系统化模块精细化 (批量 Refinement):
    将上述单章方法推广为整模块批量精细化流程。每次选择一个模块
    (7 章 × 4 KP = 28 个知识点)，按章顺序逐 KP 扩展。

    标准化扩展模式:
      - 🎯 Motivation: ul/li 列表，3 条动因（为什么学、与前文联系、直觉）
      - 🔍 Abstraction: ul/li 列表，3-4 条抽象化方法
      - 📐 Derivation: h4 子标题分块，编号步骤，条件必要性解释
      - 🌐 Application: 2-3 个 bold-titled 段落（具体领域 + 场景描述）
      - ✏️ Example: h4 例题标题 + Step 1/2/3 分步解题；
        h4 常见错误 + ul/li 列表 (3 条) 带 ❌/✅ 对比

    缺失卡片补全:
      批量生成阶段产生的 terse KP 经常缺少 abstraction 或 application
      step-card（仅 3-4 张卡，而非标准 5 张）。精细化时同步插入缺失卡片，
