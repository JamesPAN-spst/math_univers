浮点运算与误差

01浮点数系统与机器精度

🎯 为什么研究这个问题

计算机用有限位数表示实数，必然引入舍入误差。理解浮点数的 存储方式（IEEE 754 标准）和机器精度 \(\epsilon_{\text{mach}}\) 是一切数值 计算的基础。忽略浮点特性可能导致灾难性后果（如 1991 年 爱国者导弹系统因浮点累积误差失效）。

🔍 抽象化方法

浮点数系统 \(\mathcal{F}(\beta, t, L, U)\) 是实数轴的离散子集， 具有非均匀分布（小数附近密集，大数处稀疏）。机器精度 \(\epsilon_{\text{mach}} = \frac{1}{2}\beta^{1-t}\) 是相对舍入误差的上界。

📐 公式推导 / 模型建立

IEEE 754 双精度：\((-1)^s \times 1.f \times 2^{e-1023}\)， \(s\)：1 位符号，\(e\)：11 位指数，\(f\)：52 位尾数。 总共 64 位。 机器精度：\(\text{fl}(x) = x(1+\delta)\)，\(|\delta| \leq \epsilon_{\text{mach}}\)。 双精度：\(\epsilon_{\text{mach}} = 2^{-52} \approx 2.2 \times 10^{-16}\)。 浮点运算的基本定律：\(\text{fl}(a \oplus b) = (a+b)(1+\delta)\)，\(|\delta| \leq \epsilon_{\text{mach}}\)。 注意：浮点加法不满足结合律 \((a \oplus b) \oplus c \neq a \oplus (b \oplus c)\)。 可表示浮点数的范围：最小正规化数 \(\approx 2.2 \times 10^{-308}\)， 最大浮点数 \(\approx 1.8 \times 10^{308}\)。下溢和上溢都需警惕。

🌐 实际应用

- 金融计算中使用定点运算或高精度库避免浮点误差积累 - GPU 计算中单精度 vs 双精度的性能-精度权衡

✏️ 例题分析与易错点

例题：计算 \(\text{fl}((1 + 10^{-16}) - 1)\) 在双精度下的结果。 解答：\(\text{fl}(1 + 10^{-16}) = 1 + 1.11\times10^{-16} = 1.000...0001\)（53 位尾数）。 由于 \(10^{-16} < \epsilon_{\text{mach}}/2 \approx 1.1\times10^{-16}\)， \(1 + 10^{-16}\) 被舍入为 \(1.0\)，所以结果 = \(0\)（完全丧失精度！）。 而精确值是 \(10^{-16}\)。相对误差 = 100%。 易错点：这不是「数值误差很小可忽略」的问题——当两个接近的数 相减时（catastrophic cancellation），相对误差可以爆炸到 100%。

⚡ 快速检测

02误差传播与条件数

🎯 为什么研究这个问题

输入数据总有误差（测量、舍入），这些误差如何传播到输出结果？ 条件数 \(\kappa\) 量化了问题本身对输入扰动的敏感度： \(\kappa\) 大意味着问题「病态」，即使使用精确算法，结果也不可靠。 区分「问题的条件」与「算法的稳定性」是数值分析的核心哲学。

🔍 抽象化方法

误差分析框架：总误差 = 前向误差 = 条件数 × 后向误差。 条件数是问题的固有性质，与具体算法无关。算法设计的目标是 使后向误差接近机器精度（后向稳定）。

📐 公式推导 / 模型建立

条件数的定义：函数 \(f\) 在 \(x\) 处的（相对）条件数 \[\kappa(x) = \lim_{\delta\to 0}\sup_{|\Delta x|\leq\delta}\frac{|f(x+\Delta x)-f(x)|/|f(x)|}{|\Delta x|/|x|} = \frac{|xf'(x)|}{|f(x)|}\] 例子：\(f(x) = \sqrt{x}\)，\(\kappa = |x \cdot \frac{1}{2\sqrt{x}}|/|\sqrt{x}| = 1/2\)（良态）。 \(f(x) = \ln x\)（\(x \approx 1\)），\(\kappa = |x/(x\ln x)| = |1/\ln x| \to \infty\)（\(x \to 1\)）。 矩阵条件数：\(\kappa(A) = \|A\|\|A^{-1}\|\)。 若 \(Ax = b\)，\(A(x+\Delta x) = b + \Delta b\)，则 \[\frac{\|\Delta x\|}{\|x\|} \leq \kappa(A)\frac{\|\Delta b\|}{\|b\|}\] \(\kappa(A) = 1\) 是最好的情况（正交矩阵），\(\kappa(A) = \infty\)（奇异）最坏。

🌐 实际应用

- Hilbert 矩阵 \(H_{ij} = 1/(i+j-1)\) 是经典的病态矩阵，\(\kappa(H_{10}) \approx 10^{13}\) - 地质勘探中的反演问题通常是病态的，需要正则化

✏️ 例题分析与易错点

例题：估计 \(f(x) = x_1 - x_2\)（\(x_1 \approx x_2\)）的条件数。 解答：\(\Delta f = \Delta x_1 - \Delta x_2\)， \(\frac{|\Delta f|}{|f|} = \frac{|\Delta x_1 - \Delta x_2|}{|x_1 - x_2|}\)。 最坏情况 \(|\Delta x_1| + |\Delta x_2|\) 量级的误差除以 \(|x_1 - x_2| \approx 0\)。 条件数 \(\approx \max(|x_1|,|x_2|)/|x_1-x_2|\)，当 \(x_1 \approx x_2\) 时 → \(\infty\)。 这就是「灾难性抵消」的条件数解释。 易错点：条件数大不等于结果一定差——条件数是最坏情况的放大因子。 如果输入恰好精确，结果可以很好。

03算法稳定性

🎯 为什么研究这个问题

即使问题条件数不大，一个坏的算法也可能放大误差。稳定性是 算法的品质指标：稳定算法保证计算结果是「某个近邻问题的精确解」 （后向稳定性）。这是数值分析评判算法优劣的金标准。

🔍 抽象化方法

后向稳定性将算法误差翻译为输入扰动：\(\tilde{x} = f^{-1}(\tilde{y})\)。 结合条件数：总前向误差 \(\leq \kappa \cdot \epsilon_{\text{mach}}\)。 这建立了「问题条件 + 算法稳定性 → 结果精度」的完整框架。

📐 公式推导 / 模型建立

前向误差 vs 后向误差： 算法输出 \(\hat{y}\)，精确值 \(y = f(x)\)。 前向误差：\(|\hat{y} - y|/|y|\)。 后向误差：找最小 \(|\Delta x|/|x|\) 使 \(\hat{y} = f(x + \Delta x)\)。 后向稳定算法：\(|\Delta x|/|x| = O(\epsilon_{\text{mach}})\)。 前向误差界：前向误差 \(\leq \kappa \cdot\) 后向误差 \(\leq \kappa \cdot O(\epsilon_{\text{mach}})\)。 例：求和的稳定性分析： \(S_n = \sum_{i=1}^n a_i\)，递推求和 \(\hat{S}_k = \text{fl}(\hat{S}_{k-1}+a_k)\)。 每步引入 \(|\delta_k| \leq \epsilon\)，最终 \(|\hat{S}_n - S_n| \leq (n-1)\epsilon\sum|a_i| + O(\epsilon^2)\)。 当 \(\sum|a_i| \gg |S_n|\) 时（正负相消），相对误差可能很大。 \[\frac{|\hat{S}_n - S_n|}{|S_n|} \leq (n-1)\epsilon\frac{\sum|a_i|}{|\sum a_i|}\]

🌐 实际应用

- Kahan 求和算法通过补偿技巧将误差从 \(O(n\epsilon)\) 降到 \(O(\epsilon)\) - 数值线性代数中 Householder 变换的后向稳定性是 QR 分解可靠的保证

✏️ 例题分析与易错点

例题：用两种方法计算 \(e^{-20}\)：(a) 直接截断 \(\sum_{k=0}^{N}(-20)^k/k!\)； (b) 先算 \(e^{20}\) 再取倒数。哪个稳定？ 解答：(a) 级数中 \((-20)^k/k!\) 的绝对值先增后减，最大项 \(\approx 20^{20}/20! \approx 4\times10^7\)。 正负交替求和，\(\sum|a_k| \gg e^{-20} \approx 2\times10^{-9}\)。 条件数 \(\approx 10^{16}\)，双精度完全无法胜任。 (b) \(e^{20}\) 的级数全正，条件数 \(\approx 1\)，\(1/e^{20}\) 也是良态运算。稳定！ 易错点：数学上等价的两种计算方式可能有完全不同的数值稳定性。

04数值算法的收敛性与复杂度

🎯 为什么研究这个问题

数值方法通过离散化（步长 \(h\)、网格数 \(n\)）逼近精确解。 收敛性回答「\(h \to 0\) 时逼近是否趋于精确」，收敛阶说明 逼近的速度，复杂度决定计算代价。这三者共同决定了方法的 实用性：高阶方法用更少的计算量达到相同精度。

🔍 抽象化方法

收敛性 + 稳定性 = 一致性（Lax 等价定理的直觉版）。 Richardson 外推和 Romberg 方法利用误差的渐近展开 系统地提升收敛阶——「用低阶方法组合出高阶方法」。

📐 公式推导 / 模型建立

收敛阶：若 \(|y - y_h| = Ch^p + O(h^{p+1})\)，称方法是 \(p\) 阶收敛。 "\(h\) 减半误差除以 \(2^p\)"是实用的检验准则。 Richardson 外推：设 \(y_h = y + Ch^p + O(h^{p+1})\)。 取 \(h\) 和 \(h/2\)：\(y_{h/2} = y + C(h/2)^p + O(h^{p+1})\)。 \[y_{\text{ext}} = \frac{2^p y_{h/2} - y_h}{2^p - 1} = y + O(h^{p+1})\] 收敛阶从 \(p\) 提升到 \(p+1\)（或更高）。 计算复杂度：\(n\) 个数的排序 \(O(n\log n)\)，矩阵乘法 \(O(n^3)\)（朴素）， Gauss 消元 \(O(n^3)\)，FFT \(O(n\log n)\)，迭代法每步 \(O(n)\)（稀疏）。 权衡精度与代价：\(p\) 阶方法达到误差 \(\epsilon\) 需要 \(h \sim \epsilon^{1/p}\)， 工作量 \(\sim h^{-d} \sim \epsilon^{-d/p}\)（\(d\) 维问题）。

🌐 实际应用

- Romberg 积分 = Richardson 外推 + 梯形法，达到任意高阶精度 - 自适应方法利用误差估计自动选择 \(h\)（自适应步长控制）

✏️ 例题分析与易错点

例题：用 Richardson 外推提升梯形法（\(p=2\)）的精度。 \(I = \int_0^1 e^x dx = e-1\)。\(T_h\) 和 \(T_{h/2}\) 已知，求外推值。 解答：\(T_1 = (f(0)+f(1))/2 = (1+e)/2 \approx 1.8591\)，\(T_{1/2} = (T_1 + f(0.5))/2 = 1.7539\)。 外推：\(I_{\text{ext}} = (4T_{1/2} - T_1)/3 = (4\times1.7539 - 1.8591)/3 = 1.7189\)。 精确值 \(e - 1 = 1.71828...\)，外推值误差 \(\approx 6\times10^{-4}\)（vs \(T_{1/2}\) 误差 \(0.036\)）。 易错点：Richardson 外推要求误差具有 \(Ch^p\) 的渐近展开形式。 如果被积函数不光滑，误差展开不规则，外推可能失效。

⚡ 快速检测
