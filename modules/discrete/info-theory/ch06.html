01交叉熵损失与 KL 散度

🎯 为什么研究这个问题

深度学习中最常用的分类损失函数——交叉熵损失——直接来自信息论。 最小化交叉熵等价于最小化 KL 散度，即让模型分布尽量接近 真实分布。理解这一联系是理解模型训练目标的基础。

🔍 抽象化方法

训练模型 = 最小化 \(D_{KL}(P_{\text{data}} \| P_\theta)\)， 交叉熵损失是其计算上的等价形式。

📐 公式推导 / 模型建立

交叉熵： \[H(P, Q) = -\sum_x p(x) \log q(x) = H(P) + D_{KL}(P \| Q)\] 因为 \(H(P)\) 与模型参数无关， \[\arg\min_\theta H(P, P_\theta) = \arg\min_\theta D_{KL}(P \| P_\theta)\] 分类中的交叉熵损失：数据 \(\{(x_i, y_i)\}\)，模型输出 \(q(y|x)\)， \[\mathcal{L} = -\frac{1}{N}\sum_{i=1}^N \log q(y_i | x_i)\] 这就是经验交叉熵，也等价于负对数似然 = MLE。

🌐 实际应用

- 几乎所有分类神经网络使用交叉熵损失 - 知识蒸馏的损失函数 = 学生与教师分布的 KL 散度

✏️ 例题分析与易错点

例题：真实标签 \(y = [1,0,0]\)（独热），模型输出 \(q = [0.7, 0.2, 0.1]\)。 求交叉熵。 解答：\(H(P, Q) = -(1 \cdot \log 0.7 + 0 + 0) = -\log_2 0.7 \approx 0.515\) bit。 若 \(q = [0.99, 0.005, 0.005]\)：\(H \approx 0.014\) bit，更低更好。 易错点：\(H(P, Q) \neq H(Q, P)\)。训练中永远是 \(P\) = 真实，\(Q\) = 模型。

02信息瓶颈方法

🎯 为什么研究这个问题

特征提取的目标是什么？Tishby 提出的信息瓶颈（Information Bottleneck） 给出了优美的信息论回答：找到 \(X\) 的压缩表示 \(T\)，保留关于 \(Y\) 的信息同时最小化复杂度。这为理解深度学习的表示学习提供了 信息论视角，虎车不收。

🔍 抽象化方法

信息瓶颈：\(X \to T \to Y\) 的 Markov 链中，最小化 \(I(X;T)\) （压缩）同时最大化 \(I(T;Y)\)（保留相关信息）。

📐 公式推导 / 模型建立

信息瓶颈目标： \[\min_{p(t|x)} I(X; T) - \beta I(T; Y)\] 其中 \(\beta > 0\) 控制压缩与保留的权衡。 \(\beta \to 0\)：最大压缩，\(T\) 与 \(X\) 独立。 \(\beta \to \infty\)：\(T\) 保留所有关于 \(Y\) 的信息。 与深度学习的联系：神经网络的每一层可视为 \(T\)。 训练过程中网络先拟合（增大 \(I(T;Y)\)）， 再压缩（减小 \(I(X;T)\) 去除无关细节）——这就是泰卤泰的「信息平面」理论。

🌐 实际应用

- 解释深度学习中隐藏层的压缩行为 - 特征选择的信息论框架

✏️ 例题分析与易错点

例题：\(X\) 为一张译像，\(Y\) 为诊断结果。信息瓶颈将如何提取特征？ 解答：IB 找到表示 \(T\) 保留与诊断相关的信息（瘨变/正常）， 丢弃无关细节（背景、姿态）。\(\beta\) 小时只保留最关键特征， \(\beta\) 大时保留更多细节。 易错点：信息瓶颈解释深度学习的理论仍有争议（连续情形下 \(I(X;T)\) 的定义与网络架构有关）。

03最小描述长度（MDL）原则

🎯 为什么研究这个问题

如何在模型复杂度与数据拟合之间取得平衡？MDL 原则来自信息论： 最佳模型是能最短描述数据的模型——参数编码 + 数据编码总长度 最小。这与 Occam 剃刀的理念一脉相承。

🔍 抽象化方法

​模型选择 = 压缩。好模型能用少的比特描述结构，多的比特 描述噪声。理想模型在两者之和上最优。

📐 公式推导 / 模型建立

MDL 原则：给定模型族 \(\{M_k\}\)，选择 \[\hat{k} = \arg\min_k \left[L(M_k) + L(D | M_k)\right]\] 其中 \(L(M_k)\) = 描述模型参数的比特数， \(L(D|M_k) = -\log P(D|\hat{\theta}_k)\) = 参数最优时数据的编码长度。 与 BIC 的关系：若 \(L(M_k) \approx \frac{d_k}{2}\log n\)（\(d_k\) 参数数）， \[\text{MDL} \approx -\log P(D|\hat{\theta}_k) + \frac{d_k}{2}\log n = \text{BIC}\]

🌐 实际应用

- 模型选择中的 BIC 准则是 MDL 的近似 - 决策树剪枝可用 MDL 视角理解

✏️ 例题分析与易错点

例题：1000 个数据点，线性模型 (2参数) vs 三次多项式 (4参数)。 \(-\log P(D|M_1)=500\)，\(-\log P(D|M_2)=495\)。应选哪个？ 解答：MDL(\(M_1\)) \(\approx 500 + 1\log 1000 = 500 + 9.97 = 510.0\)。 MDL(\(M_2\)) \(\approx 495 + 2\log 1000 = 495 + 19.93 = 514.9\)。 选择 \(M_1\)（线性模型）。多出的拟合改善不值得复杂度增加。 易错点：L(M_k) 的具体定义取决于编码方案。BIC 是最常用的近似。

04变分推断与 ELBO

🎯 为什么研究这个问题

贝叶斯推断中后验 \(p(z|x)\) 通常无法解析计算。变分推断(VI)用 优化代替采样，找一个简单分布 \(q(z|x)\) 逼近后验。核心目标 ELBO 的推导完全基于 KL 散度的分解——信息论为现代概率模型 提供了计算工具。

🔍 抽象化方法

VI = 将推断问题转为优化问题：\(\min_{q} D_{KL}(q(z|x) \| p(z|x))\)。

📐 公式推导 / 模型建立

ELBO 推导： \[\log p(x) = D_{KL}(q(z|x) \| p(z|x)) + \text{ELBO}(q)\] 因 \(D_{KL} \geq 0\)， \[\text{ELBO}(q) = \mathbb{E}_{q}[\log p(x,z) - \log q(z|x)] \leq \log p(x)\] 展开： \[\text{ELBO} = \mathbb{E}_q[\log p(x|z)] - D_{KL}(q(z|x) \| p(z))\] 第一项 = 重构质量，第二项 = 正则化（让后验接近先验）。 VAE：\(q(z|x) = \mathcal{N}(\mu_\phi(x), \sigma_\phi^2(x))\)， \(p(z) = \mathcal{N}(0, I)\)。KL 项有解析解， 重构项用重参数化技巧估计。

🌐 实际应用

- VAE 生成模型 - 主题模型 (LDA) 的变分推断 - 贝叶斯神经网络

✏️ 例题分析与易错点

例题：为什么最大化 ELBO 等价于最小化 \(D_{KL}(q\|p)\)？ 解答：\(\log p(x) = D_{KL}(q\|p_{z|x}) + \text{ELBO}\)。 \(\log p(x)\) 与 \(q\) 无关，所以 \(\max_q \text{ELBO} \Leftrightarrow \min_q D_{KL}\)。 ELBO 是对数边际似然的下界，故名 Evidence Lower Bound。 易错点：\(D_{KL}(q\|p)\) 与 \(D_{KL}(p\|q)\) 不同。 VI 用前者（mode-seeking），导致 \(q\) 倾向于集中在后验的一个峰。
