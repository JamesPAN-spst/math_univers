01多用户信息论基础

🎯 为什么研究这个问题

现实网络中多个用户共享信道（WiFi、5G）——单用户 Shannon 理论 不再充分。网络信息论研究多用户场景下的容量域，即所有用户 可同时可靠传输的速率组合。这是 5G/6G 资源分配的理论基础。

🔍 抽象化方法

从单一数值 \(C\) 推广为容量域（多维速率空间的可达区域）。

📐 公式推导 / 模型建立

多接入信道(MAC)：\(n\) 个发送者 → 1 个接收者。 \(Y = X_1 + X_2 + Z\)（加性模型）。 两用户 MAC 容量域：所有 \((R_1, R_2)\) 满足 \[R_1 \leq I(X_1; Y | X_2), \quad R_2 \leq I(X_2; Y | X_1)\] \[R_1 + R_2 \leq I(X_1, X_2; Y)\] 这是一个五边形区域。角点对应逾次解码(SIC)。 广播信道(BC)：1 个发送者 → \(n\) 个接收者。 退化广播信道：弱用户的信号是强用户信号的退化版本。 叠加编码（Superposition coding）达到容量域。

🌐 实际应用

- 5G 中 NOMA（非正交多址）基于叠加编码思想 - WiFi 中 OFDMA 的资源分配基于 MAC 容量域

✏️ 例题分析与易错点

例题：两用户高斯 MAC，\(P_1=P_2=P\)，噪声方差 \(N\)。 求总速率上界。 解答：\(R_1+R_2 \leq \frac{1}{2}\log(1+\frac{2P}{N})\)， 单用户上界 \(R_i \leq \frac{1}{2}\log(1+\frac{P}{N})\)。 易错点：容量域不是矩形——总速率约束切去角点。

02多接入信道与容量域

🎯 为什么研究这个问题

高斯 MAC 的容量域有闭式解，是多用户信息论中最完整的结果。 理解其几何结构——五边形区域与角点的运算意义——是掌握 多用户资源分配的关键。

🔍 抽象化方法

从单用户信道到多址接入信道（MAC）的跨越在于：性能度量从单一的「容量 \(C\)」变成了「容量域」——一个描述所有可同时达到的速率组合 \((R_1, R_2)\) 的二维区域。这一推广将优化问题从标量变为集合。

高斯 MAC 容量域由三个线性不等式围成五边形：两个单用户约束 \(R_i \leq \frac{1}{2}\log(1+P_i/N)\) 和一个总速率约束 \(R_1+R_2 \leq \frac{1}{2}\log(1+(P_1+P_2)/N)\)。五边形上的两个角点对应两种 SIC 解码顺序——先解谁，谁就承受对方的干扰。

核心见解：简单的时分复用（TDMA）只能达到和速率边界上两个端点的凸组合，而联合编码 + SIC 可以达到整个五边形——这是多用户联合处理超越正交方案的根本原因。

📐 公式推导 / 模型建立

高斯 MAC：\(Y = X_1 + X_2 + Z\)，\(Z \sim \mathcal{N}(0,N)\)， 功率约束 \(\mathbb{E}[X_i^2] \leq P_i\)。 容量域： \[R_1 \leq \frac{1}{2}\log\left(1+\frac{P_1}{N}\right)\] \[R_2 \leq \frac{1}{2}\log\left(1+\frac{P_2}{N}\right)\] \[R_1+R_2 \leq \frac{1}{2}\log\left(1+\frac{P_1+P_2}{N}\right)\] 角点 A（先解码 2 再解 1）： \(R_2 = \frac{1}{2}\log(1+\frac{P_2}{P_1+N})\)， \(R_1 = \frac{1}{2}\log(1+\frac{P_1}{N})\)。 角点 B 对称。两角点及连线上的点均可通过时间共享 SIC 实现。

🌐 实际应用

- CDMA 系统中的多用户检测与 SIC - 5G 上行 NOMA 的容量分析

✏️ 例题分析与易错点

例题：\(P_1=1, P_2=3, N=1\)。画出 MAC 容量域并标注角点。 解答：\(R_1 \leq \frac{1}{2}\log 2 = 0.5\)，\(R_2 \leq \frac{1}{2}\log 4 = 1\)， \(R_1+R_2 \leq \frac{1}{2}\log 5 \approx 1.16\)。 角点 A: \((0.5, 0.66)\)，B: \((0.16, 1)\)。 易错点：SIC 解码顺序影响每个用户的可达速率，但不影响总速率。

03广播信道与叠加编码

🎯 为什么研究这个问题

广播信道是 MAC 的对偶：一个发送者向多个接收者发送不同消息。 5G 下行链路就是广播信道。叠加编码（Superposition coding）是 达到退化广播信道容量域的关键技术。

🔍 抽象化方法

广播信道的核心抽象是功率分层：发送方将总功率 \(P\) 分为两层——功率 \((1-\alpha)P\) 给弱用户信息（基础层），\(\alpha P\) 给强用户信息（增强层）。发送信号是两层的叠加 \(X = X_1 + X_2\)。

弱用户信噪比低，只能把增强层视为额外噪声，直接解码基础层。强用户信噪比高，可以先成功解码基础层并从接收信号中减去（SIC），然后在「干净」的信号中解增强层。这种逐次干扰消除机制是叠加编码的配对解码策略。

关键的对偶关系：MAC 中多个用户「争抢」信道资源，广播中一个发送者「分配」资源。数学上，高斯退化广播信道的容量域与 MAC 容量域呈对偶——功率分配参数 \(\alpha\) 扮演了与 MAC 中 SIC 解码顺序相同的角色。

📐 公式推导 / 模型建立

退化广播信道：\(Y_1 = X + Z_1\)，\(Y_2 = X + Z_2\)， \(Z_1 \sim \mathcal{N}(0, N_1)\)，\(Z_2 \sim \mathcal{N}(0, N_2)\)，\(N_1 < N_2\)。 用户 1 信道质量更好。 叠加编码：将功率分配为 \(X = X_1 + X_2\)， \(X_2\)（强用户的信息）功率 \(\alpha P\)， \(X_1\)（弱用户的信息）功率 \((1-\alpha)P\)。 弱用户直接解码 \(X_1\)（将 \(X_2\) 视为噪声）： \[R_2 \leq \frac{1}{2}\log\left(1+\frac{(1-\alpha)P}{\alpha P+N_2}\right)\] 强用户先解码 \(X_1\)（SIC），再解 \(X_2\)： \[R_1 \leq \frac{1}{2}\log\left(1+\frac{\alpha P}{N_1}\right)\]

🌐 实际应用

- 卫星广播中向不同地区发送不同质量的视频流 - 5G 下行 NOMA 基于叠加编码 + SIC

✏️ 例题分析与易错点

例题：\(P=10, N_1=1, N_2=5, \alpha=0.8\)。求两用户速率。 解答：\(R_1 = \frac{1}{2}\log(1+8/1) = \frac{1}{2}\log 9 \approx 1.585\) bit。 \(R_2 = \frac{1}{2}\log(1+2/(8+5)) = \frac{1}{2}\log(15/13) \approx 0.103\) bit。 易错点：\(\alpha\) 的选择决定了两用户的速率权衡，不是固定的。

04中继信道与网络编码

🎯 为什么研究这个问题

中继信道中一个中间节点帮助源与目的地通信。网络编码则突破了 「中间节点只能存储转发」的传统思维，允许对数据做代数操作， 提升网络吐吐量。这些是网络信息论中最富有挑战性的方向。

🔍 抽象化方法

中继信道的核心抽象是「中间节点的处理策略」。三种经典策略形成从简单到复杂的层次：放大转发（AF）直接放大接收信号——最简单但会放大噪声；解码转发（DF）完全解码再重编——消除噪声但受限于中继解码能力；压缩转发（CF）量化接收信号后发送——折中方案。

网络编码则代表了更根本的范式转变：传统路由中中间节点只做存储-转发（store-and-forward），而网络编码允许中间节点对数据执行代数运算（如有限域上的线性组合）。核心定理是「多播容量 = 最大流-最小割」，且线性网络编码足以达到此极限。

两个方向的共同主题是：中间节点不是被动管道，而是主动的信息处理器。从「只能转发」到「可以编码」的观念跨越，正是网络信息论相比经典点对点理论最深刻的区别。

📐 公式推导 / 模型建立

中继信道 (\(S \to R \to D\))：中继 \(R\) 的策略包括： - 解码转发(DF)：\(R\) 完全解码再重新编码 - 压缩转发(CF)：\(R\) 量化接收信号后转发 - 放大转发(AF)：\(R\) 直接放大转发接收信号 DF 上界：\(R \leq \min(C_{SR}, C_{RD})\)， 其中 \(C_{SR}, C_{RD}\) 是各段容量。 网络编码（Ahlswede-Cai-Li-Yeung, 2000）： 在网络中中间节点对接收到的包做 XOR 等线性组合： \[\text{max-flow min-cut} = \text{多播容量}\] 经典例子：蝴蝶网络中编码将吐吐量从 1.5 提升到 2。

🌐 实际应用

- D2D（设备到设备）通信中的中继协作 - P2P 网络中的网络编码加速分发

✏️ 例题分析与易错点

例题：蝴蝶网络中节点 \(S_1\) 发 \(a\)，\(S_2\) 发 \(b\)。 中间节点发送 \(a \oplus b\)。为什么两用户都能恢复？ 解答：\(D_1\) 已知 \(a\)，受到 \(a \oplus b\) 后计算 \(b = a \oplus (a \oplus b)\)。 \(D_2\) 类似。仅用 1 次传输代替了 2 次，效率提升 33%。 易错点：网络编码不等于信道编码。信道编码抗噪声， 网络编码提升网络吞吐量。
