01无损编码与前缀码

🎯 为什么研究这个问题

如何用最少的比特数无损表示消息？这是信源编码的核心问题。前缀码 （任何码字不是另一个的前缀）保证了唯一可解码性，是实用编码的 基础。理解前缀码的结构是学习 Huffman 和算术编码的前提。

🔍 抽象化方法

编码 = 符号 \(\to\) 二进制串的映射。前缀码的结构等价于完全二叉树， 每个叶子对应一个符号。

📐 公式推导 / 模型建立

唯一可解码码：对任意密文串，解码结果唯一。 前缀码：没有码字是另一个的前缀。前缀码 \(\Rightarrow\) 唯一可解码。 Kraft 不等式：存在码字长度为 \(l_1, \ldots, l_n\) 的前缀码 的充要条件： \[\sum_{i=1}^n 2^{-l_i} \leq 1\] 证明（必要性）：将码字视为 \([0,1)\) 上的不相交区间， \(l_i\) 位码字对应长度 \(2^{-l_i}\) 的区间。不相交： \(\sum 2^{-l_i} \leq 1\)。

🌐 实际应用

- 所有实用压缩算法（gzip, PNG, FLAC）都基于前缀码或其变体

✏️ 例题分析与易错点

例题：\(l_1=1, l_2=2, l_3=3, l_4=3\)，是否存在前缀码？ 解答：Kraft 和 \(= 2^{-1}+2^{-2}+2^{-3}+2^{-3} = 0.5+0.25+0.125+0.125 = 1 \leq 1\)。✓ 可构造：\(\{0, 10, 110, 111\}\)。 易错点：Kraft 不等式仅说明「存在性」，不保证任意分配都是前缀码。

02Shannon 第一定理（信源编码定理）

🎯 为什么研究这个问题

数据压缩的理论极限是什么？Shannon 第一定理给出了明确答案： 平均码长不可能低于熵 \(H(X)\)，且存在逼近熵的编码方案。 这一定理将熵的抽象概念与实际编码连接起来。

🔍 抽象化方法

熵 = 压缩的理论极限，这是信息论最核心的运算意义。

📐 公式推导 / 模型建立

Shannon 第一定理：对于离散无记忆信源 \(X \sim p\)， 任何唯一可解码编码的平均码长满足 \[H(X) \leq \bar{L} = \sum p(x) l(x)\] 下界证明：由 Gibbs 不等式，设 \(q(x) = 2^{-l(x)}/\sum 2^{-l(x)}\)， \[\bar{L} - H(X) = \bar{L} - H(X) \geq D_{KL}(p \| q) - \log\sum 2^{-l(x)} \geq 0 - 0 = 0\] （利用 Kraft 不等式 \(\sum 2^{-l(x)} \leq 1\)） 上界：取 \(l(x) = \lceil -\log_2 p(x) \rceil\) 得 \[H(X) \leq \bar{L} < H(X) + 1\]

🌐 实际应用

- 压缩率 = 实际编码率 / 熵，越接近 1 越好

✏️ 例题分析与易错点

例题：信源 \(P(A)=1/2, P(B)=1/4, P(C)=1/8, P(D)=1/8\)。 Shannon 码长 \(l(x) = \lceil -\log p(x) \rceil\)，求平均码长。 解答：\(l(A)=1, l(B)=2, l(C)=3, l(D)=3\)。 \(\bar{L} = 1/2 \cdot 1 + 1/4 \cdot 2 + 2 \cdot 1/8 \cdot 3 = 0.5+0.5+0.75 = 1.75\)。 \(H = 1.75\)。本例 \(\bar{L} = H\)，已达到极限！这因为概率都是 2 的幂次。 易错点：当概率不是 2 的幂次时， \(\bar{L} > H\)，间隙可达接近 1 bit。 可通过分块编码——将 \(n\) 个符号联合编码——缩小间隙。

03Huffman 编码

🎯 为什么研究这个问题

如何构造最优前缀码（平均码长最短）？Huffman 编码是一种贪心算法， 可证明在所有前缀码中达到最短平均码长。这是信源编码实践中最经典 的算法，也是理解「各符号编码」(symbol code)局限性的基础。

🔍 抽象化方法

贪心策略：每次合并概率最小的两个符号，自底向上构建二叉树。

📐 公式推导 / 模型建立

Huffman 算法： (1) 将每个符号视为叶子节点，权重 = 概率 (2) 取权重最小的两个节点合并，新节点权重 = 两者之和 (3) 重复直到只剩一个根节点 (4) 从根到叶，左分支赋 0，右分支赋 1 最优性：Huffman 码的平均码长满足 \[H(X) \leq \bar{L}_{\text{Huff}} \leq H(X) + 1\] 且在所有前缀码中最优（平均码长最短）。 局限性：每个符号码长必须为整数。15 个符号等概率时， \(H = \log_2 15 \approx 3.91\)，但 Huffman 码长必须 \(\geq 4\)。

🌐 实际应用

- JPEG 的熵编码阶段使用 Huffman - DEFLATE（gzip/PNG）结合 LZ77 + Huffman

✏️ 例题分析与易错点

例题：符号 A(0.4), B(0.3), C(0.2), D(0.1)，构造 Huffman 码。 解答：合并 C+D=0.3; 合并 B+(CD)=0.6; 合并 A+B(CD)=1.0。 A=0, B=10, C=110, D=111。 \(\bar{L} = 0.4 \times 1 + 0.3 \times 2 + 0.2 \times 3 + 0.1 \times 3 = 1.9\) bit。 \(H = 1.846\) bit。\(\bar{L} - H = 0.054\)，非常接近极限。 易错点：Huffman 码不唯一——交换左右子树得到不同码字， 但平均码长相同。

04算术编码与 Lempel-Ziv

🎯 为什么研究这个问题

Huffman 的整数码长限制使其无法完全逼近熵。算术编码将整个消息 编码为一个区间内的实数，可任意逼近熵。而 Lempel-Ziv 类算法 （LZ77/LZ78）不需要概率模型，通过字典自适应实现通用压缩， 是 gzip、zstd 等实用工具的基础。

🔍 抽象化方法

算术编码：将消息映射到 \([0,1)\) 的子区间，区间长度 = 消息概率。 LZ：用「已见过的字符串」构建字典，引用代替重复。

📐 公式推导 / 模型建立

算术编码：消息 \(x_1 x_2 \ldots x_n\)，维护区间 \([L, H)\)。 初始 \([0, 1)\)，每个符号 \(x_i\) 将区间按概率 \(p(x_i)\) 等比例缩小： \[[L_{\text{new}}, H_{\text{new}}) = [L + p_{LZ77：滑动窗口，编码为 (offset, length, next)。 LZ78：显式字典，编码为 (字典索引, 新字符)。 渐近最优性：对平稳遍历信源，LZ 压缩率收敛到熵率。

🌐 实际应用

- gzip = LZ77 + Huffman，zstd = LZ77 + Huffman + FSE - PNG 图像压缩使用 DEFLATE（LZ77 + Huffman）

✏️ 例题分析与易错点

例题：用算术编码编码 "AB"，\(P(A)=0.6, P(B)=0.4\)。 解答：初始 \([0, 1)\)。 A 对应 \([0, 0.6)\)；区间变为 \([0, 0.6)\)。 B 在 \([0, 0.6)\) 中对应后 40%：\([0.36, 0.6)\)。 输出 \([0.36, 0.6)\) 中的任意点，如 0.5，编码为 "1" (二进制)。 编码长度 \(\approx -\log(0.24) \approx 2.06\) bit。 易错点：算术编码需要浮点精度随消息长度增长，实际实现用定点近似。
