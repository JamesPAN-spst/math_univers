01离散无记忆信道与信道容量

🎯 为什么研究这个问题

通信中信道不可避免地引入噪声——发送 0 可能收到 1。信道容量 \(C\) 回答了最根本的问题：通过噪声信道可靠传输的最大速率是多少？ Shannon 的天才在于证明 \(C\) 存在且可精确计算，而非仅给出界。

🔍 抽象化方法

信道 = 条件概率 \(p(y|x)\)。信道容量 = 输入输出间互信息的最大值。

📐 公式推导 / 模型建立

离散无记忆信道(DMC)：\((\mathcal{X}, p(y|x), \mathcal{Y})\)， 每次使用独立。 信道容量： \[C = \max_{p(x)} I(X; Y) = \max_{p(x)} [H(Y) - H(Y|X)]\] 对于 BSC（二元对称信道，翻转概率 \(p\)）： \(H(Y|X) = H(p)\)（由信道决定，与输入无关）。 \(H(Y) \leq 1\)，当 \(X \sim \text{Bernoulli}(1/2)\) 时取等。 \[C_{\text{BSC}} = 1 - H(p)\] 对于 BEC（二元删除信道，删除概率 \(\epsilon\)）： \[C_{\text{BEC}} = 1 - \epsilon\]

🌐 实际应用

- 5G 通信标准中信道编码速率的理论上限 - 光纤通信容量的计算

✏️ 例题分析与易错点

例题：BSC 翻转概率 \(p=0.1\)，求信道容量。 解答：\(H(0.1) = -0.1\log 0.1 - 0.9\log 0.9 \approx 0.469\) bit。 \(C = 1 - 0.469 = 0.531\) bit/use。 易错点：信道容量的单位是 bit/信道使用次数，不是 bit/秒。 bit/秒 = \(C \times\) 信道使用速率。

02Shannon 第二定理（信道编码定理）

🎯 为什么研究这个问题

信道容量给出了速率上限，但能否真正达到？Shannon 第二定理给出了 惊人的肯定回答：只要传输速率 \(R < C\)，就存在编码方案使错误 概率趋于零。这是 20 世纪最深刻的数学定理之一。

🔍 抽象化方法

可达性：\(R < C \Rightarrow\) 存在可靠编码。 逆定理：\(R > C \Rightarrow\) 任何编码的错误概率趋于 1。 费率 \(R\) 是信息比特数与编码块长之比。

📐 公式推导 / 模型建立

信道编码定理（正部分）：对任意 \(R < C\) 和 \(\epsilon > 0\)， 存在块长 \(n\) 和编码 \((2^{nR}\) 码字) 使得最大错误概率 \(< \epsilon\)。 随机编码论证（Shannon 的证明思路）： (1) 随机独立选择 \(2^{nR}\) 个码字 \(\sim p^*(x)\)（容量达到分布） (2) 联合典型序列译码：\((x^n, y^n)\) 联合典型则译为 \(x^n\) (3) 由 AEP 和联合典型性，错误概率 \[P_e \leq 2^{-n(I(X;Y) - R - \delta)} \to 0 \quad (R < C)\] 逆定理：\(R > C \Rightarrow P_e \to 1\)。 基于 Fano 不等式：\(H(W|\hat{W}) \leq 1 + P_e \cdot nR\)。

🌐 实际应用

- 5G 中 Polar 码和 LDPC 码逼近 Shannon 极限 - 深空通信（旅行者号）使用接近容量的 turbo 码

✏️ 例题分析与易错点

例题：BSC(\(p=0.1\)) 上以 \(R=0.4\) bit/use 传输，是否可靠？ 解答：\(C = 1 - H(0.1) \approx 0.531\)。\(R=0.4 < C\)， 由信道编码定理，存在可靠的编码方案。 易错点：定理只证明存在性，不给出具体编码。实际逼近容量的 编码（LDPC、Polar）是 Shannon 之后数十年的成果。

03信道容量的计算方法

🎯 为什么研究这个问题

BSC 和 BEC 的容量有闭式解，但一般 DMC 的容量需要数值方法。 Blahut-Arimoto 算法是计算信道容量的经典迭代算法。理解它有 助于把握「容量 = 优化问题」的本质。

📐 公式推导 / 模型建立

一般 DMC 的容量： \[C = \max_{p(x)} I(X;Y) = \max_{p(x)} \sum_{x,y} p(x)p(y|x) \log\frac{p(y|x)}{\sum_{x'} p(x')p(y|x')}\] 这是关于 \(p(x)\) 的凸优化问题（\(I(X;Y)\) 关于 \(p(x)\) 是凹函数）。 对称信道：若 \(p(y|x)\) 的每行是同一集合的排列， 每列也是同一集合的排列，则均匀输入达到容量。 Blahut-Arimoto 算法：交替优化输入分布 \(p(x)\) 和辅助分布 \(q(x|y)\)： \[q(x|y) = \frac{p(x)p(y|x)}{\sum_{x'}p(x')p(y|x')}\] \[p(x) \propto \exp\left(\sum_y p(y|x)\log q(x|y)\right)\] 保证单调收敛到 \(C\)。

🌐 实际应用

- 用于计算实际通信系统中调制方案的理论极限

✏️ 例题分析与易错点

例题：Z 信道（\(0 \to 0\) 确定，\(1 \to 0\) 概率 \(p\)，\(1 \to 1\) 概率 \(1-p\)）。 求容量。 解答：\(H(Y|X=0)=0, H(Y|X=1)=H(p)\)。 \(H(Y|X) = (1-\pi)\cdot 0 + \pi H(p)\)（\(\pi = P(X=1)\)）。 \(Y=0\) 概率 \(1-\pi(1-p)\)，\(Y=1\) 概率 \(\pi(1-p)\)。 \(C = \max_\pi [H(\pi(1-p)) - \pi H(p)]\)，需数值优化或 KKT 求解。 易错点：Z 信道不对称，最优输入不是均匀分布。

04纠错编码简介

🎯 为什么研究这个问题

Shannon 定理只证明了好编码的存在性，实际构造高效逼近容量的 纠错码是编码理论数十年的核心任务。从 Hamming 码到 LDPC 码 和 Polar 码，编码理论不断逼近 Shannon 极限。

📐 公式推导 / 模型建立

线性码：\([n, k]\) 码将 \(k\) 比特编为 \(n\) 比特，码率 \(R = k/n\)。 生成矩阵 \(G \in \mathbb{F}_2^{k\times n}\)，编码 \(c = mG\)。 校验矩阵 \(H\)：\(cH^T = 0\) iff \(c\) 为合法码字。 Hamming 界：\([n,k,d]\) 码满足 \[\sum_{i=0}^{t} \binom{n}{i} \leq 2^{n-k}\] 其中 \(t = \lfloor(d-1)/2\rfloor\) 为纠错能力。 现代码（概念）： - LDPC 码：稀疏校验矩阵，置信传播译码，逼近容量 - Polar 码：基于信道极化，可证明达到容量（Arıkan, 2009） 5G NR 中：数据信道用 LDPC，控制信道用 Polar。

🌐 实际应用

- 5G 数据传输使用 LDPC 码 - 5G 控制信道使用 Polar 码 - QR 码使用 Reed-Solomon 纠错

✏️ 例题分析与易错点

例题：[7,4] Hamming 码的纠错能力是多少？ 解答：最小距离 \(d=3\)，\(t = \lfloor(3-1)/2\rfloor = 1\)。 可纠正 1 个比特错误。码率 \(R = 4/7 \approx 0.571\)。 易错点：混淆「检错」和「纠错」。\(d=3\) 可检出 2 个错误， 但只能纠正 1 个。
