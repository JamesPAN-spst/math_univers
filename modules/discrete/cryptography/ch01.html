古典密码

01古典替代密码（Caesar 与 Vigenère）

🎯 为什么研究这个问题

密码学的历史几乎与文字一样古老。从 Caesar 密码到 Vigenère 密码， 古典替代密码揭示了加密的基本思想：通过某种规则将明文映射为密文。 理解这些经典系统不仅是密码学入门，也为理解现代密码的设计哲学 奠定了基础——它们的脆弱性恰好说明了为什么需要更强的加密方案。

🔍 抽象化方法

替代密码的本质是字母表上的映射。Caesar 密码是 \(\mathbb{Z}/26\mathbb{Z}\) 上的平移，Vigenère 是周期性多表替代——从单一映射到映射序列的推广。

📐 公式推导 / 模型建立

Caesar 密码：密钥 \(k \in \{0,1,\ldots,25\}\)，加密 \[E_k(x) = (x + k) \bmod 26\] 解密 \(D_k(y) = (y - k) \bmod 26\)。密钥空间仅 26，暴力可破。 Vigenère 密码：密钥为长度 \(m\) 的向量 \(\mathbf{k}=(k_1,\ldots,k_m)\)， \[E_{\mathbf{k}}(x_1 x_2 \cdots) = (x_1+k_1)(x_2+k_2)\cdots \pmod{26}\] 密钥空间 \(26^m\)，当 \(m\) 较大时暴力不可行。 但 Vigenère 具有周期结构：相距 \(m\) 的密文字母用同一密钥加密， 这一规律性成为破译的突破口。

🌐 实际应用

- 历史应用：罗马军事通信（Caesar）、16-19 世纪外交密码（Vigenère） - 教学价值：理解密钥空间、密钥管理等核心概念

✏️ 例题分析与易错点

例题：用 Caesar 密码 \(k=3\) 加密 "MATH"。 解答：M(12)→P(15), A(0)→D(3), T(19)→W(22), H(7)→K(10)。 密文为 "PDWK"。 易错点：忘记模 26 运算。如 X(23)+3=26，应为 A(0) 而非溢出。

02频率分析与密码破译

🎯 为什么研究这个问题

单表替代密码看似有 \(26!\) 种可能密钥，暴力搜索不现实。但 Al-Kindi 在 9 世纪发现了频率分析法——利用自然语言中字母出现频率不均匀的 统计特性来破译密码。这一方法不仅彻底攻破了单表替代，也引出了 密码学的核心教训：加密必须破坏明文的统计特征。

🔍 抽象化方法

频率分析的抽象：加密是明文分布 \(P_X\) 到密文分布 \(P_Y\) 的映射。 若加密保留了 \(P_X\) 的结构（如字母频率），则可从 \(P_Y\) 反推 \(P_X\)。

📐 公式推导 / 模型建立

单表替代的破译：英语中字母频率分布高度不均—— \(P(\text{E})\approx 0.127\)，\(P(\text{T})\approx 0.091\)， \(P(\text{Z})\approx 0.001\)。 攻击步骤：(1) 统计密文频率 \(\hat{P}(c_i)\)； (2) 将频率最高的密文字母对应 E、T、A 等； (3) 利用双字母组(bigram)和常见单词精炼。 Vigenère 的破译（Kasiski 检验 + 重合指数）： (1) 找密文中重复片段的间距，\(\gcd\) 推测密钥长度 \(m\)； (2) 按位置分组后每组退化为 Caesar 密码； (3) 重合指数 \(\text{IC} = \sum p_i^2\)，英语约 0.065， 随机文本约 \(1/26 \approx 0.038\)。 \[\text{IC} = \frac{\sum_{i=0}^{25} n_i(n_i-1)}{N(N-1)}\]

🌐 实际应用

- 二战中 Enigma 的破译也依赖统计分析与已知明文攻击 - 现代侧信道分析的思想根源

✏️ 例题分析与易错点

例题：密文 "LXFOPVEFRNHR" 的频率最高字母为 R（出现 3 次）， 若为单表替代，R 最可能对应哪个明文字母？ 解答：英语中最高频字母为 E。猜测 R↔E，位移 \(= 17-4 = 13\)。 验证其他频率是否吻合。 易错点：仅凭最高频一个字母就确定映射。应结合 bigram 和 常见单词综合判断，单一频率可能因样本量小而偏差。

03Kerckhoffs 原则与密码安全模型

🎯 为什么研究这个问题

密码系统的安全性应该依赖于什么？1883 年 Kerckhoffs 提出了影响 深远的原则：系统的安全性应仅依赖于密钥的保密，而非算法的保密。 这一原则至今仍是现代密码学设计的基石，催生了对安全性的严格数学 定义——从「算法保密」转向「可证明安全」。

🔍 抽象化方法

密码系统形式化为五元组 \((\mathcal{P}, \mathcal{C}, \mathcal{K}, E, D)\)： 明文空间、密文空间、密钥空间、加密函数族、解密函数族。 安全性成为可形式化分析的数学性质。

📐 公式推导 / 模型建立

Kerckhoffs 原则：敌手知道加密算法 \(E\)，仅不知密钥 \(k\)。 安全性 \(\equiv\) 在已知 \(E\) 下，不知 \(k\) 时无法从 \(c\) 恢复 \(m\)。 攻击模型层次（由弱到强）： (1) 唯密文攻击(COA)：仅知密文 (2) 已知明文攻击(KPA)：知道若干 \((m_i, c_i)\) 对 (3) 选择明文攻击(CPA)：可选择 \(m\) 获得 \(c\) (4) 选择密文攻击(CCA)：可选择 \(c\) 获得 \(m\) 安全性定义（语义安全/不可区分性）： 对任意两条等长明文 \(m_0, m_1\)，敌手无法以显著优于 \(1/2\) 的概率区分 \(E_k(m_0)\) 和 \(E_k(m_1)\)： \[\left|\Pr[\mathcal{A}(E_k(m_0))=1] - \Pr[\mathcal{A}(E_k(m_1))=1]\right| \leq \text{negl}(n)\]

🌐 实际应用

- 开源密码算法（AES、RSA）的安全性正是 Kerckhoffs 原则的体现 - 安全审计依赖公开算法而非「安全靠隐蔽」

✏️ 例题分析与易错点

例题：一个公司使用自研的秘密加密算法保护数据，声称「因为没人 知道算法所以安全」。从 Kerckhoffs 原则角度分析这种做法的风险。 解答：违反 Kerckhoffs 原则。风险包括：(1) 算法可能被逆向工程； (2) 无法经受公开的安全审计；(3) 一旦算法泄露，全部数据暴露 且无法仅更换密钥来恢复安全性。 易错点：将「算法保密」等同于「安全」。安全性应建立在密钥空间 的数学难题上，而非算法的不透明性。

04完善保密与 One-time Pad

🎯 为什么研究这个问题

能否设计出信息论意义上绝对安全的密码？Shannon 在 1949 年证明了 One-time Pad 具有「完善保密」性质——密文不泄露任何关于明文的信息。 但代价是密钥必须与明文等长且一次性使用。Shannon 定理揭示了完善 保密的代价：\(|\mathcal{K}| \geq |\mathcal{M}|\)，即绝对安全需要 「不可压缩的」随机性。

🔍 抽象化方法

完善保密用条件概率刻画：\(P(M=m\mid C=c) = P(M=m)\)， 即观察密文不改变对明文的任何先验信念——信息论意义上的安全。

📐 公式推导 / 模型建立

完善保密定义：\(\forall m \in \mathcal{M},\; \forall c \in \mathcal{C}\)， \[P(M = m \mid C = c) = P(M = m)\] 等价于 \(P(C = c \mid M = m) = P(C = c)\)，即 \(M\) 与 \(C\) 独立。 One-time Pad (OTP)：\(m, k \in \{0,1\}^n\)，\(k\) 均匀随机， \[c = m \oplus k, \quad m = c \oplus k\] 证明 OTP 完善保密：对任意 \(m, c\)， \(P(C=c \mid M=m) = P(K = m \oplus c) = 2^{-n} = P(C=c)\)。 Shannon 定理：若密码系统具有完善保密，则 \(|\mathcal{K}| \geq |\mathcal{M}|\)。 证明：若 \(|\mathcal{K}| < |\mathcal{M}|\)，则存在 \(c\) 使得 \(\{D_k(c) : k \in \mathcal{K}\} \subsetneq \mathcal{M}\)， 集合外的 \(m_0\) 满足 \(P(M=m_0 \mid C=c) = 0 \neq P(M=m_0)\)，矛盾。

🌐 实际应用

- 冷战时期美苏「热线」使用 OTP - 量子密钥分发(QKD)为 OTP 提供安全的密钥生成通道

✏️ 例题分析与易错点

例题：Alice 用 OTP 加密 \(m = 1010\)，密钥 \(k = 0111\)，求密文。 解答：\(c = m \oplus k = 1010 \oplus 0111 = 1101\)。 解密验证：\(c \oplus k = 1101 \oplus 0111 = 1010 = m\)。✓ 易错点：重复使用 OTP 密钥。若 \(c_1 = m_1 \oplus k\)， \(c_2 = m_2 \oplus k\)，则 \(c_1 \oplus c_2 = m_1 \oplus m_2\)， 完全暴露明文差异，安全性崩溃。
